2.14 vectors(向量)

在c\c++中没有向量类型，如果想要使用必须自己定义， 比如数组， 事实上在c\c++中向量是一组标量整列而已， 因为gpu直接处理向量类型所以gpu内部内建了向量类型，当然cg也直接支持了向量类型。

注意事项：

在cg中的向量类型，如float3和float4是不同的， 而且数组float x[4]和float4也是不同的，向量类型是一个阵列数据， 这种阵列数据在处理2个向量操作时，使用一个gpu指令就可以完成，叉乘， 加， 点乘

    这种快速操作在c\c++不能被支持，虽然现在的cpu指令， 如Intel的 SSE2, SSE, and MMX; AMD的3DNow!， 和Motorola的 AltiVec指令加入了向量操作， 但是这些指令不能被传统的编程语言支持，但是cg对向量操作提供了特别支持， cg编译器提供了gpu可编程控制器的高速向量操作

Cg有许多特性， 这取决于你选择的cg  profile文件， 向量通常有四个或者少于四个分量，cg提供了极高的效率向量操作，比如赋值， 取负， 求绝对值， 乘， 加， 点乘， 叉乘。

另外一方面， 使用非常量索引访问向量可能低效或者不能支持， 这依赖于profile， 例如：

2.2.2 cg编译错误

  Cg程序一般有两类编译错误：传统的编译错误， profile依赖错误

  传统的编译错误一般无非是程序编写上问题，这种错误和c\c++的编译错误一样的处理办法，

    Profile依赖错误主要是使用cg是你profile不能支持你的cg程序，这种错误不是一般传统程序有的。

2.2.3     profile依赖错误

    profile依赖错误通常会引起3d编程接口和你使用的gpu硬件，这里有

三类依赖错误， 兼容性错误， 语境错误， 和硬件支持错误。

    兼容性错误

目前所有的片段着色profile都可以操作纹理， 但是所有的顶点profile却不能，原因很简单，目前gpu顶点着色器不能支持纹理操作， 但是未来可能支持

Cg不允许编译一个不能执行一个profile不支持程序，如果一个顶点profile不支持纹理操作， 但是程序却试图去执行，那么兼容性错误就会出现。

语境错误

这是一种基本错误，尽管稀少， 例如，一个顶点程序没有输出一个参数绑定给POSITION语境，这种错误时应为图形管道假定所有的顶点都有一个位置，

同样， 一个片段程序不能返回一个ＰＯＳＩＴＩＯＮ，　总的来说，这些错误都是使用ｃｇ方式和图形管道数据流不一致造成的，

硬件支持错误

这种错误主要是由于ｇｐｕ兼容性造成的，　有些ｇｐｕ不能在一个通路中支持４个材质，　但是有些ｇｐｕ在一个通路中可以支持任意数量的材质，严格的说，由于ｇｐｕ种类众多，并不是所有的ｃｇ指令都能被这些ｇｐｕ支持，这往往产生错误

硬件支持错误往往令人沮丧，　因为这种错误不能通过检查源码来纠正，比如你可能给一个顶点程序指令使用太大的数而造成溢出，　但是这种错误却很难查找。

预防错误

有两条路可以避免错误， 1是尽量使用高级profile， 使用高级profile你可以避免兼容性错误， 越是使用高级profile，随着可编程图形硬件更行，这将减少兼容性错误。

2是你要尽可能的了解兼容性，语境和硬件支持错误，翻阅cg文档可以了解这些限制，

你可以经常在使用新的3d接口时在profile依赖限制有错觉，参照你的opengl和direct3d文档， 这会帮你定位profile依赖限制

      到目前为止， 我们的例子只涉及到顶点程序，这节我们将展示一个简单的片段程序，他能很好的和我们的顶点程序配合。

Example 2-2. The C2E2f_passthrough Fragment Program

struct C2E2f_Output
{
  float4 color : COLOR;
};
 
C2E2f_Output C2E2f_passthrough(float4 color : COLOR)
{
  C2E2f_Output OUT;
  OUT.color = color;
  return OUT;
}

 这个比C2E1v_green的顶点程序简单的多， 实际上他几乎什么都没有做，

程序的输出没有改变光栅阶段的色彩， gpu光栅硬件直接使用这个色彩去程序帧缓冲区。

下面是C2E2f_passthrough的输出结构

struct C2E2f_Output {
  float4 color : COLOR;
};

片段程序一般比顶点程序的输出简单程序， 例子中接受一个4分量向量， 名字叫“name”， COLOR 输入语境是光栅化差值得到的， 并且基于图元装配阶段指定的色彩。

C2E2f_passthrough的函数体是这样的

{
  C2E2f_Output OUT;
  OUT.color = color;
  return OUT；
}

在声明一个C2E2f_Output 类型大的OUT变量后， 我们的程序仅仅将硬件差值的颜色值直接给OUT数值，然后程序将这个值返回
