斑马骑士介绍

Pamakids Tech，英文名是“爸爸、妈妈和孩子们”的意思，斑马骑士，是音译名称。我们觉得斑马是非洲大地上一种有趣而神秘的动物，它黑白分明，很难说它是黑马还是白马？我们希望它是白马王子和黑马王子的结合体。而骑士，象征着侠义、勇敢、忠诚和荣誉，全面发展的“骑士七艺”（游泳、投枪、击剑、骑术、狩猎、弈棋、诗歌）。骑士精神，仿佛中国的武侠、侠客，有本领，讲道德侠义，保护弱小。我们希望斑马骑士是一群保护我们的未来-孩子的大人，为孩子考虑，为孩子设计。

斑马骑士科技有限公司，是一家专注于儿童早期智能教育的科技创业公司。目标通过创意、设计和技术，提供给孩子和父母最专业、最好的教育产品，去激发儿童探索和创造的潜质。基于对心理学、教育学的系统认识和对触摸屏、传感器等交互设计的深刻理解，我们致力以寓教于乐的方式让孩子参与互动操作，充分调动孩子的感官能力和兴趣，来引导他们主动学习。让父母以最省力的方式进行教育管理，有更多的时间精力去陪伴和关爱孩子。

团队特点：

  1. 体验与教育驱动的产品：对孩子，斑马骑士产品通过精良的体验设计让孩子主动“参与”到认知学习中，去激发并追踪他们的兴趣与热情；对父母，斑马骑士是去帮助他们方便和愉悦地管理孩子的教育活动，帮父母节省时间去更多的陪伴孩子并与他们交流。
  2. 基于iOS/Android平台的Pad/Phone或其它终端（如SmartTV）：目前斑马骑士的产品主要基于iOS和Android平板或手机，也可以开发基于智能电视的应用，给孩子和父母提供基于先进的教育理念和出色用户体验的软硬件产品。
  3. Air跨平台技术的早期探索者：斑马骑士是国内最早尝试基于Adobe的Air for mobile和Starling（Stage3D GPU加速引擎）技术开发移动终端应用/游戏的团队，积累了丰富的经验，是Adobe中国地区少数深度合作的技术团队之一。充分发挥跨平台技术和模块化开发的优势，提高了开发效率降低了开发成本。同分辨率iOS游戏移植到android只需要几分钟时间。
  4. 跨界和丰富的经验：团队成员的背景涵盖心理学、教育学、艺术设计、游戏设计、软件工程等。大多数团队成员有5年以上相关领域的经验，半数来自创新工场，积累了丰富的前瞻性研究、软硬件设计、触摸屏/移动终端交互设计、游戏设计运营等经验。

《鲸鱼岛的冬天》介绍

苹果App Store 6·1首页第一位推荐，教育付费榜第一名，付费总榜第八名。大自然的神奇魔法，孩子指尖上的想象！最具“交互性和趣味性”的“原创”儿童教育游戏！

斑马骑士公司出品，供3-9岁孩子自主学习自然百科常识的互动教育游戏。

原创故事梗概

鲸鱼岛是一个没有冬天的美丽小岛。岛上的居民从来没有经历过冰雪的冬天。小猪波波最大的愿望就是过一个白色的冬天，于是他许下了一个愿望。小朋友，月亮姐姐会教给你大自然的魔法，来帮小猪波波完成他的心愿。

独特产品理念

基于心理学和教育学的研究，斑马骑士团队探索了人类在游戏中探索、学习的方式，将天气变化等自然知识融入到游戏的操作中，让孩子在玩耍和游戏中自然而然的了解并理解关于冬天的天气、活动的知识。

创意交互体验

App Store第一款充分利用了iPad的各种交互性的游戏，创新地将触屏手势、多点触控、重力加速度、声音控制等技术有机地融入到游戏中，让孩子用有趣的方式体验各种操作，并不断探索和学习。

开发过程中的5个对的事情

  1. 选对了开发技术

    由于开发团队多为webgame专业背景，更熟悉actionscript，所以在团队开发之初，在多种开发平台综合比较好，选择了air。这是我们最明智的选择之一，air让我们的美术和程序开发做到无缝链接，提高了我们开发效率，使得我们可以将更重要的精力放在游戏创意与用户体验上。并且因为air的跨平台开发，确保了我们将游戏产品轻松的配置到其他的移动平台上。应用了adobe air支持的全部ipad2上的交互技术。

    adobe 对air的投入让我们更放心的选择这个技术，同样的程序用air2.7发布只能运行38fps，用air3.3发布60fps。 在我们开发过程中，也一直能得到adobe方面的指导与建议，鲸鱼岛的开发过程中我们遇到过一个的问题，当我们启用麦克风的监时，发现麦克风所录制的音乐会实时的被播放出来，这个bug会影响用户的体验，就在我们无计可施的时候， James Li推荐我们用ane满足需求实现。

    选用Starling框架，刚开始选用Stage3D框架的时候，准备从Genome2D，Starling,以及ND2D中选择一个。经过一周多的熟悉及测试，最后定下Starling框架，主要是基于几个原因。第一个原因就是开源，可以做到最大程度的把控，一切都是有文档可查和容易理解的，学习成本可以降到最低。第二个就是熟悉的 API设计，很容易上手。Starling是一个纯粹的ActionScript3库，模仿传统的Flash显示列表架构，和传统的显示对象不同的是，starling的所有内容都直接提供给GPU,渲染性能较之前有很大的提升。相对来说ND2D也是开源的，API也不是太复杂，当时放弃选用ND2D主要是因为鼠标事件不太稳定。Genome2D应该算是效率最高的框架了，但是不开源，复杂的API设计足以让人望而生畏。最后一个原因，就是后期维护支持问题了，Starling是Adobe官方维护的这样作为开发者就可以放心的使用了。开始使用starling的时候还是2月份，那时候虽然starling功能还不是太完善，但是从项目本身出发以及开源的特点，足以支撑项目了。因此就选择了starling,并一直用到现在，基本公司主要产品的开发都是基于starling框架的。

  2. 选对了开发技术

    插件式架构设计

    之所以采用插件式设计，主要是《鲸鱼岛的冬天》的游戏类型决定的。基于插件设计的好处很多：把扩展功能从框架中剥离出来，降低了框架的复杂度，让框架更容易实现。扩展功能与框架以一种很松的方式耦合，两者在保持接口不变的情况下，可以独立变化和发布。简单的说是为了让设计更简单，更容易理解。《鲸鱼岛的动天》是以交互和时间为驱动的，通过不同的交互和时间变化会触发不同的结果。就涉及到多模块的通信和管理。一个插件下面可以存在几个不同的模块，整个游戏由若干个插件构成，支持即插即拔式管理。消息的广播传递通过一个消息控制器，分发给每个消息对应的接受方，从而简化流程，提高程序的运行效率。测试某一个模块的时候，只需要发送一条消息就可以了，方便测试及代码的管理。

package{ public interface IPlugin{ function init():void; function release():void; function update():void; function getId():int; function OnBroadcastMsg(msg:MsgBase):void } } package { public class MsgBase { public var plugins:Array = []; } } package { public class PluginControl { static function start():Boolean static function getPluginByID(pluginId:int):IPlugin; static function registerPlugin(plugin:IPlugin):Boolean; static function logoutPlugin(pluginId:int):void static function broadcastMsg(msg:MsgBase):void } }

    选择用TexturePacker生成动画

    矢量素材在移动设备中的运行效率不是很高，我们需要将这些动画全部转换成图片，那么TexturePacker更好的帮助管理和生成图片的材质，首先我们从flash中将动画逐帧的导出成图片，然后通过TexturePacker将很多张小图片组装集合成一张大的材质集合，同时TexturePacker还可以生成是starling的配置文件，方便我们在程序中读取和加载。现在flashcs6也有同样的功能，比TexturePacker更佳方便的完成此类操作。

  1. 探索儿童自然学习的方式，站在儿童视角设计游戏

    自然学习的方式，在中国的观念里，学习与游戏的功能是截然分开的，然而在观察孩子的过程中，我们发现自然状态下真正的学习来自与孩子们带着兴趣玩游戏时想要解决问题，想要玩起来的主动探索中，学习与玩并不矛盾。如果在玩的过程中，有动手尝试，参与游戏的环节，主动探索，顿悟的学习过程，会让孩子在玩的过程中，更有动力，更容易激发孩子的创造和想象，获得学习过程中的成就感，学到的知识和技能也会更牢固。

    举一个例子，我们在餐馆里吃饭时，给一个4岁小女孩玩pad上的游戏，她被adobe一个工程师设计的“ Sylvester’s band “所吸引，尽管她还不认识上面的英文，却乐在其中，会主动尝试pad的各种交互功能来完成任务，她坐在桌前身体俯着用手指头点着pad，当海狸在森林里寻找乐队成员时，海狸跑得不够快，她开始摆弄pad，经过各种尝试，偶然端起让pad倾斜着，发现小猪就能跑得快起来，经过多次尝试发现了这个规律。可是她更好奇这个规律的原因，便向我们提问，经过一番演示和讲解，她明白原来是怎么回事。之后只要遇到类似的问题，她都会这样尝试着解决。

    另外，我们将鲸鱼岛的冬天定位在绘本级的应用，绘本设计精美、信息量非常大，孩子获取信息是分年龄层的，过一个阶段重新翻阅，依然会有新的收获，因此孩子会反复阅读。而在成人游戏或界面设计时，需要限制过大的信息量。

  2. 刚刚好的优质设计

    常见的为青少年、成人提供的网络游戏或在线休闲游戏，在设计上非常精美华丽，而面向儿童的游戏，在中国会以比如数学、语言、英语、电子书等课程教育相关内容为主，游戏开发大多追求批量快速，开发者也会忽略儿童对人物、场景、动画、界面等方面的视觉要求，因此在设计上相对简单一些，往往一个星期就能出一款儿童产品。

    与孩子家长想的一样，在物质上供给孩子最高品质的物质生活，鲸鱼岛团队也认为应当给予儿童最高品质最合适他们的包括电子书、玩具、游戏等精神消费。按照这个理念，团队制定了更高的设计标准，首次为3-6岁儿童设计人物角色、游戏场景、和游戏界面时，我们从头开始做基础调研，体验同类儿童app产品的设计，翻阅大量童书与儿童绘本，去儿童幼儿园和画室了解儿童喜欢的风格。

    鲸鱼岛的设计师小强，在设计过程中，沿袭了他过去在成人游戏中精致的画风，丰富鲜明的人物角色和动作动画。并没有因为做儿童产品有丝毫的放松，挑选出不同风格的卡通视频、童书、儿童绘本，与大家讨论，以不同风格的手绘草图，与大家碰撞。

    不仅是画面的设计，在游戏音乐和音效上，我们也觉得儿童产品需要更精细的设计。在游戏配乐上，我们找了专门的音乐人进行创作编曲，才有了鲸鱼岛现在生动好听的主题音乐。而语音配音和音效上，我们也是精心设计和挑选。开始找了2个专业的动画片配音演员，但觉得没有妈妈给孩子讲故事的亲切和生动的感觉，于是，我们果断放弃，找了一个幼儿园的音乐老师进行录音，才有了现在“鲸鱼岛”开场时的小故事。

  3. 为孩子设计为中心的教育理念支持

    常见的游戏通常以第一人称视角，扮演英雄或救世主去拯救世界。但我们想也许不一定只有自己成功才能获得快乐。鲸鱼岛的冬天，要求玩游戏的小朋友以帮助游戏主角的方式进行游戏。于是，“鲸鱼岛”才有了一个第三人称的视角，小朋友是以帮助小猪波波的方式进行游戏，在游戏的过程中，小猪波波的愿望此起彼伏的出现，小朋友通过摸索和演示等方式，不断得尝试，在帮助他人的过程中获得游戏的快乐。

    我们接收到的用户反馈也支持了这点，很多大人在下载了鲸鱼岛的冬天后，有些不知所措，而他们的小朋友却很自然得玩起来。

开发过程中的5个问题

我们在开发鲸鱼岛冬天的过程中，几经周折，这里将我们用血泪换来的经验与大家分享。希望各位其他开发同行能少走弯路，提高开发效率。

  1. 测试！ 测试！ 测试！

    测试流程，尽可能早的进行真机测试

    由于我们的开发环境是在电脑，电脑的运行效率很高，所以有一些问题在电脑中不会呈现出来的，建议大家每发布一个版本模块的时候都放到移动设备上进行测试。尽早的发现问题，逐个击破，同时还得积累在移动端的代码书写规范，为以后的开发积累经验。

    急于发布，测试偏少，导致crash问题

    在鲸鱼岛的最后发布过程中，由于发布心切，忽视了测试的作用，提交之后才发现crash的问题，便被appstore驳回，错过了发布后的运营与推广的机会。这里建议大家在提交之前多多测试。回想当时的情景，如果当时我们提前做好了充分的测试，那就能成为第一个在appstore上线的adobe aire的产品了！现在我们只能是第二个。

  2. 重视ios版本更新与用户的使用习惯

    关注ios版本更新，规范操作

    苹果在更新5.0.1后，加入了icloud云存储的功能，ios会认为存放在document目录下的文件都是需要运同步的，由于我们没有及时的了解更新说明，将大量的资源文件装载到document，再次被appstore审核驳回，教训惨重。

    不要一次性高频率的加载过多的材质

    在游戏的运行中，我们会在游戏开始的初期将游戏的资源文件加载到内存中，以便保证游戏在后面更佳流畅的运行。我们在开发的时候经常发现会在这个阶段app异常的退出了。经过测试发现：在ios设备中不可以高频率的去加载过多的材质，我们可以通过延迟加载来确保，整个加载工程中不会出现异常退出的情况。同时提示一下，ios的用户群体中是非常反感在应用运行过程中退出的情况。

  3. 儿童游戏与教育分类定义模糊，对家长需求的把握要准确

    鲸鱼岛的冬天，产品里虽有很深的教育理念和意义，但形态上家长容易认为它是一款游戏，而对于游戏，家长，尤其是中国的父母，对于电子游戏是有天然的担心和戒备的。上传鲸鱼岛的冬天时，我们也不知道应该应对在app store的哪个分类里更合适些。虽然在游戏说明里做了强调，但效果甚微，如果面对面向家长说明这款app的意义时，用户还是会非常认同。可这样直接面对终端用户的机会并不多。这也是我们发现在鲸鱼岛上线后，虽然业内好评如潮，但下载量一般般的原因。

    这需要我们做更多功课提高产品给用户的直观感受上，找到更有效说服家长的方式，说明产品对孩子的作用和意义，以及孩子的乐趣。

  4. 错过了关键时机

    季节

    鲸鱼岛的冬天，本来是希望新年上线的，赶在冬天的尾巴。但由于第一次用Air和Stage3D进行游戏的开发，也由于对评估审核机制的不熟悉，上线时，冬天已经基本过去，春天已经来了。所以，新年的各种促销的机会都没有抢上。提醒大家，要规划充分的创意和开发时间。

    赶上了app store的繁忙期

    虽然我们赶上了2012年6月1日儿童节，并且被app store的编辑关注，上了编辑推荐和网站首页大banner，还很荣幸得在6月1日当天在apple专营店内做产品宣讲。但实际上在上传鲸鱼岛的时候，正好是apple召开一年一度新品大会的日子，自然也与伟大的乔布斯一周年祭撞了车。虽然斑马骑士团队对这位伟大的人物怀着非常崇敬的心情，也完全能理解编辑们都得转精力筹备大会，但上传之后，超长的时间没有得到编辑的反馈，我们心中还是多少有些怨念的。

  5. 中国app游戏市场

    最后一点，也是最重要的一点。鲸鱼岛是斑马骑士团队的第一个游戏产品，我们也是第一次接触关于app运营和营销的事情。还没有太多的运营经验，只是根据我们的直觉去做了一些比如新闻发布会、参加各种app论坛、开设@斑马骑士微博等工作，但没有了解做在中国运营app，必须更了解中国用户，以及中国app运营的大市场环境。尤其是在中国用户更愿意下载破解版本时，开始对我们来说是不小的打击。这也是我们目前特别放精力在关注和尝试的部分，也希望其他的打算面向中国市场开发和投放儿童游戏app的团队特别注意的。


设计

鲸鱼岛的创意源自一个模拟自然呼风唤雨的疯狂想法，最初就想能把自然界的风霜雨雪照搬到一个盒子大小的世界里，于是鲸鱼岛的设计从一开始就被注入了非真实世界的血统。听起来很酷吧？做起来头疼死了！

早期是没有鲸鱼概念的，所设计的世界都是有边界的：各种形状的盒子、瓶子、沙盘等等等等，所有能想到的能在桌面上模拟世界的容器被想了一个又一个。既要能看清楚盒子里面的世界， 又要能是用户在小小的屏幕上操作不至于太费力。那段时间用绞尽脑汁四个字形容在形象不过了。突然有一天好像顿悟了：ipad本来不就是个盒子么？为什么我们还要在盒子里再放个盒子？于是，我们把ipad的屏幕当作一个盒子来布置我们的世界，而我们就好像从家里的窗户里去看另一个。在这个世界里是有水的，当然也有陆地。就像一个古老美丽的童话故事一样：在茫茫的大海上，有一个美丽的岛屿，上面种满了高大翠绿的树，一栋温馨的小木屋，上面住着快乐岛民。

想着就觉得很酷，很快一个水天一色之间一座黄色的岛屿漂浮起来。一次调侃中，觉得这个黄色的大家伙看着好像一只即将浮出水面的鲸鱼先生。诶！多好的想法！为什么不可以？。。。好吧，鲸鱼岛就这么诞生了！

总的来说最初的鲸鱼岛看起来有些像一个用糖果堆积起来的世界。那么安静，那么恬美，但是又那么脆弱，好像一阵风就会把她吹碎掉一样。整个世界里看上去没有什么生机，像个圣地一样，让人向往又让人敬畏，生怕打破这个安静的气氛。可是这完全不是我们先要的效果啊？我们虽然是想让他梦幻些，但是又那么的触手可及，那么的想真的走进去。

好吧，改版！

为了不让鲸鱼岛那么镜花水月，决定把她立体起来。之前的鲸鱼岛世界虽然恬美，但是看上去有些平面，我宁愿相信她是根据一个传说描绘出来的拼贴画，并不是那么的真实。这次把鲸鱼岛世界整个转个一个方向，改过去的平视为现在的略带俯视。并且给岛上的房子、树、水和木头都加上了纹理材质，让他们看上去更真实立体。为了避免他们看上去像一个沙盘上的立体玩具，采用了手绘的风格。轻轻的扫上几下笔触，把这些立体的家伙们从真实的世界里拽回来。

这次改版最突破的可能要算是两个天空巨头：太阳和月亮了。

之前的太阳和月亮用的都是真实的太阳和月亮。也是出于不让这个世界更像真实沙盘游戏的照片，这次才用了欧洲古老宗教经常会用到的太阳脸和月亮脸。在哥特式教堂巨大的玻璃彩绘上，在欧洲古老的百年家族的纹章里，这两个图案太常见了。圆圆的太阳脸，一张庄严并略带微笑的脸，象征着愉悦，幸福，成功和满足。弯弯的月亮脸，一张慈母般微笑的脸，象征着成长，吸收和积累。这两个古老的图案放在这个游戏里，既象征着我们对参与这个游戏小朋友们的美好祝福，又不失对这两个神圣的boss的尊敬和敬畏。

还有，这次改版的配色方案也是一个比较巨大的突破。

之前鲸鱼岛的配色，包括其他公司的同类产品的配色，大多都是以同色系颜色为主，很少有强对比色的使用，美其名曰：保护孩子视力，不能使用过激的颜色对比。颜色是可以用来影响和表达情绪的。如果孩子一直都接触这种影响，眼前永远都是那些很粉嫩，很缓和的颜色，女孩子倒还好，受其影响的男孩子会不会很娘？。。。。

所以我决定打破这个怪圈，适当的加入对比色和加大对比度。当然，这不代表这个游戏是不可爱的，还有很多可爱的元素在里面，例如那只胖胖的，有着上翘的嘴角的可爱鲸鱼先生，和圆滚滚的喜欢滑冰的猪先生。

整个的鲸鱼岛的设计不止倾注了所有设计人员的心血，而是全部斑马骑士团队共同努力的结果。这款设计代表了斑马骑士团队对于儿童产品设计方面的方向，虽然不那么透彻，但是我们会继续诠释下去。我们不甘于不动脑筋的跟随儿童设计的大潮流，我们希望努力描绘出真正多姿多彩，有着丰富的元素的儿童世界。这也是斑马骑士团队在视觉方面的设计底线，只为孩子设计。

我们在提交app store审核时，正好赶上苹果的大会和Jobs的周年祭，估计所有的app store的编辑都在忙着这两件大事，导致大洋彼岸的我们苦等。着急的我们多次邮件询问，也还是反馈很慢，团队成团一起通宵等待发布的那个夜晚非常难忘。

斑马其实选择air的原因：

  1. Air高效率的开发效率，让美术和程序无缝的链接，让我们讲更多的时间投入到产品创意上和用户体验上。
  2. Air跨平台开发，完成一个项目后，我们可以轻松的将这个产品适配到大多数移动平台上。
  3. 我们的团队不会objectC，我们也不会java，我们最擅长的语言就是actionscript。
  4. 同样的程序用air2.7发布只能运行38fps，用air3.3发布60fps，adobe 对air的投入让我们更放心的选择这个技术。
