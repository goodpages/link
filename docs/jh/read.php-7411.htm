为什么要开始写作

如果你要从一堆人中决定出一个职位的合适人选，雇那个写作最厉害的人。这个人不在乎是不是营销人员、推销员、设计师、程序员，无论是谁；他们的写作技巧会对此有益。

—— Rework


记得在大学的时候，JustPub 创始人周筠老师来我们团队做了一次分享。分享的主题是关于写作，她鼓励我们每个人都建立一个博客，记录和分享自己的想法。但作为一个标准的理科生 + 前程序员，写作一直是我的软肋，所以博客也就断断续续的写过一两篇。


参加工作后，作为一个喜欢选择 Hard 模式的玩家，我转行做了产品经理 —— 每天的工作从写代码切换到了写文档。渐渐地，我发现无论何种职业，都避不开写作，它对于思考的意义和价值远大于我的想象。

 写作是更系统的思考

1. 记录思维轨迹，便于回溯和反思

我们都有过这样的经历：通电话时，对方告诉你需要联系另一个人，并提供了他的电话号码。但此时你手边并没有笔纸来记录，只好一遍一遍的默念这个号码来帮助记忆，然后在挂了电话以后立刻拨打。如果不巧在这个时候有同事来找你，你多半会很恼火，并且可能会忘记刚才记录的号码。


心理学家管这种需要不断维持，记忆时长不超过一分钟的记忆方式叫做工作记忆（working memory）。工作记忆的一个重要特点是它极为有限，稍纵即逝。


这就导致人在思考一个问题的时候，就像是在黑暗中打着电筒往前走。每一步推导都将我们往前挪一小步，然而电筒的光亮能照到的范围是有限的，向前走了一段再回头就发现看不见自己走过的路线了。这也是为什么在思考的过程中，我们经常想到后面的就忘了之前的，想到某个分支就忘了另一个分支，而一旦想岔了，岔得越远，越难回到当初岔开的点。


所以如果我们仅凭记忆进行逻辑推导，将会容易遗忘思维轨迹。通过写作可以很好的避免这个问题。在写作过程中，推导每向前一步，都会将阶段性成果转化为文字并记录下来，提醒我们之前思考的每一个岔路口，即使一时忘记，也总归可以回到最初岔开的店。这便为我们回溯思维轨迹提供了可能性。

2. 避免碎片化，形成系统的思考

根据我自己的经验，有时候当遇到一个新问题时，经过一段时间的思考，我会认为自己已经思考的比较全面和深入了。但当有机会和不同人聊起这个问题时，就会发现我给出的结论都不太一样。


这是因为交谈的环境是放松的舒适的，在潜意识里，我们知道对方可能并不会去深究每一个细节,或者反驳语言中的漏洞。所以在这个时候，人脑中本能的一部分会承担主要的思考工作，进行无序而碎片化的思考。


更糟糕的是，由于「本能的思考」具备联想的能力，这使得交谈过程中，不时有「好的想法」从脑子里蹦出来，并让我们产生对问题认识更深刻的错觉。这时候会出现想到哪说到哪的情况，正式这种非常碎片化的思维，使得我们每次交谈都容易给出不太一样的结论。


如果我们要培养系统的思维方式，就需要学会将自己的思考写下来。在写作过程中，我们经常会遇到一时语塞无法下笔的情况。这说明我们对当前问题思考是不够透彻的，我们缺少足够有信服力的论据，当强迫自己继续写下去，并要求自己明确而富有逻辑地表达自己的观点，其实意味着需要做更多额外的工作：围绕这个问题，大量查阅资料，补充论据，同时强迫自己的大脑「用力地思考」，进而整理形成一条清晰的逻辑线。


你会发现，写作过程本身便是一种刻意的系统思维的训练方式，它强制你将碎片化的思想整合为一个具有信服力的思维框架。

 写作是更高效的分享

长久以来，在不同的情境下，面对不同的朋友，我都习惯于通过交谈的方式来分享自己在一段时间内的认知和体验。久而久之我发现，每次讨论所需的时间成本是固定的，而且该成本并不会因熟悉度的提升而降低。这显然是一种效率比较低的分享方式。


换句话说，交谈是一种杠杆率比较低的分享方式。什么是杠杆率？格鲁夫在《格鲁夫给经理人的第一课》中提出了管理杠杆率的概念，他认为可以用以下的公式来衡量管理活动的产出：


管理的产出 = 杠杆率 * 管理活动


类似的，我们可以用杠杆率来衡量分享方式的效率：


分享的产出 = 杠杆率 * 分享方式


同时格鲁夫给出了提升杠杆率的方法：注重活动的时间效率，在单位时间内影响更多的人。所以如果通过一种分享方式，在影响的人数增加的同时，你付出的劳动量或者时间随之增加，说明这种方式的杠杆率是比较低的。反之，如果随着影响人数的扩大，你付出的劳动量并没有增加，甚至劳动成本下降为零，那么这种分享方式就具备很高的杠杆率。


写作正是杠杆率很高的分享方式。因为一篇文章在完成后，借住互联网的传播渠道，能够零成本的分享给很多人阅读和交流。这是一种成本固定，影响力却能够无限扩大的分享方式。因此，如果你乐于分享自己的认知，将它写下来，能搞极大的提高效率，扩大你的分享价值和影响力。


写作是一件知易行难的事情，但我必须训练自己闭上嘴巴，将思考写下来。
