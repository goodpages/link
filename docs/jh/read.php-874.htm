在Internet上收集了一些编译器或解释器作品供诸位同学参考，尽管某些作品还存在一些缺陷，无法作为商业化编译器使用，但它们都提供了完整的编译器源代码（可惜文档甚至程序注释往往不充分），对学习编译技术有重要参考价值。以下样板作品已按所处理的语言以及作品的功能进行了分类。请参考这些样板作品的设计、源程序、文档，但千万不要抄袭它们！

记号：╠ S → T(m) ╣P表示以P语言编写的、将S语言转换为T语言的编译器，m表示可执行T语言的虚拟机或解释器（若T没有一个明确的名字，则以虚拟机或解释器m代替语言T）；若T部分为空，则表示以P语言编写的S语言解释器；若T为特定处理器平台的机器语言，则记为◇。

所有样板作品已经分为3类：A类表示各种玩具语言（Toy Languages）的实现，所谓玩具语言通常是为学习编译器构造而从某类实用语言中简化和抽象出来的语言；B类表示各种实用语言的实现，主要集中在Pascal、C/C++、Java、BASIC等语言；C类表示虚拟机（或编译后端仿真工具）、编译器生成工具的实现。
