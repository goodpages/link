第四章 坐标转化

      当你写顶点或者片段程序， 你必须理解坐标系统是怎么工作的， 这一章解释图形管道发生了什么，当然这些知识我们不能详细解释数学基础了， 这章涉及两个主题

      坐标系统， 解释坐标系统怎么将模型顶点坐标进行转换的

      理论实施， 解释运用矩阵和坐标系统

4.1 坐标系统

      图形管道的作用是建立和在屏幕上显示图片，图形管道使用几何数据来建立2坐标图片， 你程序提供几何数据（比如多边形的顶点）， 屏幕的图像一般就是摄像机看到的东西了。

      当几何数据通过管道， gpu顶点处理器会把顶点数据转到一个或者多个不同的坐标系统， 每个转换都有其特殊用处，cg顶点程序提供你自己的方式转化数据。

      顶点程序也可以处理别的文物， 如灯光和动画， 但是顶点转化使其重要的工作， 你不可能写一个不需要顶点转化的程序，因为光栅化时需要转化后的顶点， 这样才可以图元装配生成面片。

      迄今为止，我们的程序都是关于2d方面的，本章解释如何进行3d坐标转换。

      图4-1展示如何进行顶点转换， 并且说明如何从一个转化阶段到下个阶段过程

Figure 4-1 Coordinate Systems and Transforms for Vertex Processing

      下面主题将会解释上面的各个转换阶段，我们假定你有基本的数学知识， 并且我们将会从较高的层次讲解一些概要知识。

4.1.1 物体坐标

      程序在坐标系统制定顶点位置，这个坐标就是物体坐标， 当美工做好了模型， 他们会在物体坐标系中设置每个模型顶点适当的位置，方位， 有时2个物体的物体坐标没有任何联系， 比如， 一个圆柱可能设置在物体坐标系的中心并且依z坐标对称。

      你需要描绘出每个顶点在物体坐标系，作为一个顶点， 你的程序维护他们的位置， 或许还有法线之类的。

4.1.4      齐次坐标

     更一般的， 我们可以将位置<x,y,z>表达为<x,y,z,w>格式，这种表达叫做齐次坐标，当我解释位置坐标为<x,y,z>时，其实我们默认w为1

      在数学上， w值一般都会被x,y.z相除， 如图4-1所示

     Equation 4-1 Converting Between Nonhomogeneous and Homogeneous Positions

      将位置表示为齐次坐标有好多好处， 第一，多重变换，如3d投影视图需要的投影矩阵，这些操作使用4x4矩阵进行更有效率， 这些技术将会4-2节中解释，特别是做透视时的投影除法。

      4.12 世界空间

           物体空间提供了一个独立于其他物体的3维空间，世界空间为所有的物体提供了一个绝对位置，如何建立一个世界坐标系统是任意的， 例如， 你可以将你的世界坐标系的原点设置为你房子的中心，房子中的物体相对于屋子的中心，物体的方位和坐标的向上方向都可以按照你的意愿设置。

      4.1.4 模型转换

      对于物体空间中的点转化为世界空间中的点的过程我们称作为模型转换，例如， 你需要旋转， 移动，和缩放一个桌子模型，这样桌子才可以放在房子所在世界坐标系的合适位置，在同一个房子中的2个桌子可能使用相同的3d模型，但是使用不同的模型转换，这样桌子就可以放在2个不同的位置。

      在本章我们使用4x4矩阵来表示所有模型转换，根据矩阵的性质，你可以将旋转， 移动， 缩放矩阵并且投射到一个单独的4X4矩阵。当你使用这种方式连接矩阵， 这个联合矩阵代表着所有的变换，这个变换非常强大，待会儿你就会看到。

      如果你使用4X4矩阵来表示模型转换， 转化的结构就是将物体转化到世界坐标，相同的数学原则将会在后面的一系列转换中继续使用。

右边的图将机器人的不同部分做了不同的转换的样子，

      4.1．5 视点空间

      最终， 你想得到的是从视点方向看的图像，这个坐标系统中我们称之为视点空间， 视点坐落为坐标系的原点，在标准转换下，你面对的场景时，眼睛开的方向为z轴， 向上的方向一般是y轴。

      视点空间对光照很有用，我们将会在第五章讨论

4.1．6 视图转换

      这个转换将会把世界空间的位置转换到视点空间， 这次你将会再次使用4x4矩阵。

      典型的视点转换联合了眼睛在世界坐标系中的移动操作和旋转操作， 为了处理这些，视点转换定义位置和视点的方位。

      图4-3展示视点转换， 左边的图展示了图4-2机器人和眼睛， 他的位置在世界坐标中《0,0,5》，右边的图展示视点空间，坐标的原点在坐标原点， 在这个例子中，视图转换把机器人移动到视图空间的正确的位置， 等到转换结束， 机器人的位置为《0,0，-5》，

当眼睛在原点。 视点空间和世界空间分享y轴的正方向为向上的方向，并且移动只在z轴上进行。旋转和移动类似。

Figure 4-3 The Effect of the Viewing Transformation

      模型视图矩阵

      很多光照和一些顶点运算都涉及位置和法线， 这些运算在视点空间和物体空间更有效率，世界空间对于所有物体的相对位置计算很方便，但是对于光照和顶点运算效率比较低。

      由于这个原因，我通常将模型和视点转换合成你一个矩阵这就是我们所有的模型视图矩阵， 你可以使用普通的乘法来合成矩阵

      4.1.7 裁剪空间

     当一个位置在视点空间， 下一步是判断这个位置是否应该显示，那就需要一个坐标系统去裁剪，这个坐标就是裁剪作坐标。

      Cg顶点程序中输出的positon就是裁剪空间的位置，每一个顶点程序可以随意输出纹理坐标和颜色，但是顶点程序只会输出一个裁剪空间坐标

      4.18 投影转换

      这个转换就是将视点空间坐标转到裁剪空间坐标， 这个转换就是我们所说的投影转换。

投影转换定义了一个平头的视椎体来代表摄像机看到的区域，当渲染成图像时至于多边形，点，线才是潜在可视的。Opengl和direct3d在处理裁剪空间上有轻微的差别，在opengl上任何可以看到的物体都会放在x，y，z轴对其的立方体中，但是要保证-w x w, -w y w, and -w z w.不等式得到满足， direct3d和opengl类似，但是必须保证可视物体在z轴上满足0 z w，这些规则说明在裁剪空间中所有的点都是以齐次坐标形式表示。

      投影转换提供查询这个标准投影立方体的方法，一般来说我们将这个转换表示为4x4矩阵。

      投影矩阵

      图4-4展示投影矩阵如何将在视点空间中的机器人转化到裁剪空间。这个机器人正好适应在裁剪空间中，而且最终图片也会将裁剪空间中的机器人完整绘制出来。

Figure 4-4 The Effect of the Projection Matrix

      裁剪规则在opengl和direct3d是不同的，cg程序根据你使用api需要依赖合适的投影矩阵， 这两种裁剪空间区别不是那么明显， 你的程序需要提供合适的矩阵给你的cg程序。

4.1.9      归一化设备坐标

     裁剪坐标使用齐次坐标，但是我们需要计算带有深度值的2d点， （深度值是为了深度缓冲准备的， 而深度缓冲区为渲染可视的面提供硬件访问方法）

      投影除法

      用x，y，z除以w可以完成投影除法， 这个最总的坐标就是所谓的归一化设备坐标，

所有的可视几何数据在opengl中将会限制在<-1, -1, -1> 和<1, 1, 1> 之间, 在direct3d中限制在 <-1, -1, 0> and <1, 1, 1> 之间

      在2~3章中的2d顶点程序就是所谓归一化设备坐标，在这些例子中假定z值是0，w介于0~1之间。

4.1.10   窗口坐标

最后一步设备坐标转化为窗口坐标，一般的，顶点程序接受物体空间的顶点坐标，程序将会使用模型视点矩阵和投影矩阵将设备坐标转换为裁剪坐标，实际上，我们将会将2个矩阵合并为一个。 图4-5展示转换的2中方式

Figure 4-5 Optimizations for Transforming to Clip Space

例子4-1将是将物体坐标转到裁剪坐标

Example 4-1. The C4E1v_transform Vertex Program

void C4E1v_transform(float4 position  : POSITION,
 
                 out float4 oPosition : POSITION,
 
             uniform float4x4 modelViewProj)
 
{
 
  // Transform position from object space to clip space
 
  oPosition = mul(modelViewProj, position);
 
}

这个程序使用物体坐标的点并且将模型视点投影矩阵作为输入，你的程序需要提供这些数据， 在本书中我们假定输出参数都有o的前缀
