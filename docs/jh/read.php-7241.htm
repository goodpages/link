是不是该转型搞机器学习呢？

renjihe1988 , 21:30 , 置顶 , 评论(0) , 引用(0) , 阅读(3) , Via 本站原创 大 | 中 | 小

里面提到这样一些统计数据：

  1. 卡内基梅隆大学（也称 CMU），是美国计算机专业最顶级的大学之一，更是很多机器学习大牛的发源地。其研究生课程《Introduction to Machine Learning》的注册人数在过去的五年里增到了六倍。而申请到 CMU 攻读机器学习博士学位的申请人在过去六年里翻了一倍。

  2. 类似的增长在很多别的大学都可以看到。伯克利大学（另一个 top 大学）本科生课程《Introduction to Machine Learning》的注册人数两年增到了三倍。伯克利大学申请人工智能方向方向的学问的申请者也在五年里翻了一倍。

  3. 伯克利大学的 EECS 申请者里面有大约 34％ 是申请人工智能方向。


这些其实相当一部分和最近 Deep Learning 越来越火有着密不可分的关系。DL 最近在无人驾驶，图像识别，实时翻译，围棋等等方面的成功，让越来越多的人相信，DL 可以在医疗、能源、金融、娱乐等等方面都能有很好的应用。因此，也有越来越多的人开始往这个专业看齐。


而 Google、Facebook、Microsoft、IBM 以及一些中小公司也开始招聘一些机器学习的专业人士。


也开始有很多人去开始思考要不要花时间自己学习下机器学习？要不要选一个机器学习的学位来攻读一下？已经在职又觉得自己的领域没什么意思的也考虑是不是可以转型到搞机器学习？


其实很多事情都是因人而异的。我也只能就我自己和一些周围朋友的经验以及硅谷我所能了解的一些情况来谈一谈普通的机器学习专业的人才，在就业方面可能会遇到的一些情况吧。（如果你是达到了能够深度理解 AlphaGo 这样的人才，下面说的可能就不适用了）。


学的人多并不代表门槛低


机器学习炒的火热，如果仅仅是了解一些皮毛，大家聊天的时候可以搭个话，别人做项目跟你讨论的时候你能知道他吐槽的决策树、神经网络是什么，提到的训练集、各种误差之间是什么关系你都了解他在说什么，这并不很难。我之前在 MacTalk 上分享的一篇 《如何学习一门技术》便是试图用最白话的方式尽可能的介绍机器学习中一些最基本的概念（链接：https://goo.gl/SK8KlN）。对于有一定基础的朋友可能自学一门入门的课程或是参加一个培训，几个月就能有所小成，对机器学习有一定的了解。


然而，要想真的能够对机器学习有比较深入的了解并运用到实际问题中，一定要对其中的各种算法真正自己推导了解。因为机器学习的理论涉及到很多门学科，如概率论、统计学、逼近论、凸分析、计算复杂性理论等。没有一定的数学底子，说实话其实学起来会步步维艰。可能有些人轻松就懂了的地方，有些人却只能花大量的精力先来啃一些数学书才能吃透。


举个例子，机器学习里最基本最入门的一个关于 Gradient Descent （梯度下降法）的计算，了解其概念并不难，但是试试照着维基百科或者别的教材的介绍推导一次，看看是不是真的弄明白，可能对自己学习这个领域需要的精力投入就能有个大概的体会。


获取机器学习职位，需要达到什么水准


这个问题其实想了解一点都不难。你找个 Google、Facebook 之类的招聘主页，看看他们对这个职位的技能要求就知道了。我们看看 Google 是怎么要求的：



其他公司的要求其实是很类似的，基本都是（1）需要一个硕士或以上的专业学位，或者/以及（2）你有工作经验证明你做出一个靠谱的机器学习的系统，并且用于实际产品中。


这仅仅是你的简历会被 recruiter 留中的前提条件，现在学机器学习的也很多，更是不乏名校正规学历的和有着实战经验的人。就算你拿到了面试机会，面试中，你可能被问到如下的问题（参见 Quora：What are some common Machine Learning interview questions?）


  • 挑一个你熟悉的算法并解释他的数学含义。然后用伪代码实现。

  • 挑另一个更高级的算法解释其数学，并伪代码实现。

  • 什么是 ROC 曲线，写出生成该曲线的伪代码

  • 讨论你对特征工程的理解以及相关技术。


如果你是有机器学习工作经验的，可能还会被问到：


  • 你怎么选择、收集、和清理数据？

  • 在你的项目中你是怎么选择特征的？

  • 你用了什么算法/模型，为什么？怎么评估？

  • 你觉得和别的算法相比优缺点是什么？

  • 你遇到的困难？怎么克服的？


当然这仅仅是例子。各个公司问的题目都有区别，但是可以看到，取得这样一份机器学习的工作，需要你对很多现有机器学习的算法和数学有很深的理解；实际动手搭建过系统，对各种工具都比较熟悉，并知道机器学习实际应用中常见的问题以及该如何处理。这也是为什么很多入门级的机器学习的同学连这样的职位的面试机会都拿不到。


在职的机器学习的工程师和数据科学家


尽管机器学习和深度学习被炒的热火朝天，其实很多中小公司的机器学习的应用并没有如大家想象的那样成熟。这里面原因有很多。可能是公司没有招到领域里真正有实力有经验的人把整个系统搭起来。可能是公司的数据或者资源匮乏，机器学习并不能发挥其期许的成效。可能是重视程度不够，所以各种投入不够，训练的算法一直是最简单的，上线后更新也不频繁……等等。所以机器学习在很多中小公司仅仅对一些比较成熟的算法和模型的重用。在这样的模型上做一些调整等，其实大部分时候是工程而不是机器学习了。这也是为什么很多人慢慢有一种错觉，好像在搞机器学习的工程师或者数据科学家，也并不需要特别高深的机器学习知识。


而对于一些机器学习的应用已经做的比较成熟的公司，或者是有能力走在技术最前端的公司，却又是另外一种情形。这样的团队通常会致力于做出适合其产品需求的模型或算法改进、完整的训练和评估系统、以及相关的大数据的采集和处理的数据管道等等。因此，其技能集起于机器学习，却又不限于机器学习。最核心的成员需要对系统和机器学习均有很好的把握和相当的经验。而能做到这一点，可以说在如今对机器学习人才求贤若渴的今天，你的职场竞争力可以完全秒杀大部分码农了。然而大部分人，可能还是仅仅做一些相关的实现和工程。


所以结论是什么？


我说的并不一定都是对的，仅仅是和一些朋友讨论关于 “在职的人跟风换方向到机器学习有没有前途” 的一些共识：

  1. 机器学习和深度学习炙手可热的今天，如果还刚刚大学毕业，确实可以考虑读个相关的学位，练练级。如果几年后你毕业形势还这么好。那么可以肯定，你的就职选择肯定会很广的。

  2. 如果你想中途出家，机器学习的门槛其实还是蛮高的。除非你仅仅是想多了解而不求精通，或者你各种数学底子很扎实。否则你不可能像你学习另一门语言一样使用业余时间来轻松搞定机器学习。你必须花出足够的精力和恒心，因为这并不是一门可以轻松转型的技术方向。

  3. 作为一项技术来了解，还是很有益有趣的。新的对全栈工程师的定义，兴许都会把对机器学习的基本了解作为标准之一。（谁知道呢？全栈工程师的定义从来就没有达到过共识。）

  4. 我的朋友里，也不乏 “曾经是搞机器学习方向的，如今也仅仅是把自己当机器使” 的朋友。机器学习的就业形势，除了一些做出成就的牛牛，说老实话我并没看出比一般工程师多出太多的优势。
