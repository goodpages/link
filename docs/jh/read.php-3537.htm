频繁出镜的企业家，风险投资的热钱，甚至“范式转移”（Paradigm Shift），这些迹象正预示着开源盛世的到来。

软件企业家约翰·牛顿（John Newton）是Documentum公司的共同创始人。2003年，牛顿和公司的高级管理层以17亿美元的价格，将Documentum公司卖给了EMC公司。Documentum公司是当时全球最优秀的内容管理软件供应商之一。根据目前的估算，内容管理领域的市场规模将会达到每年25亿美元。现在，牛顿决定挑战自己，重新再来一次。


在出售了Documentum公司后，牛顿并没有走向开曼群岛（Cayman Islands，它和英属维尔京群岛一样，吸引了很多准备在纳斯达克股票市场上市的公司在这里注册空壳企业，运作上市），而是重新回到了自己早期成功的地方，来证明他完全可以东山再起。牛顿的策略，印证了软件市场正在发生的巨大变化，尤其对于创业公司而言，更是如此。2005年6月，牛顿和前博奥杰公司（Business Objects S.A.）首席运营官（COO）约翰·鲍威尔（John Powell）创立了Alfresco软件公司，提供内容管理软件，公司的核心技术是由开源社区许多有独立思想的程序员设计的。通过巧妙发挥这些编程人员的智慧，Alfresco公司希望打破内容管理市场为EMC公司、Interwoven公司、Vignette公司等主流厂商所把持的的局面。


自从网络泡沫破灭以来，我们还从未听到过创业公司敢于发表如此大胆的言论。无论是从积极方面来看，还是从消极方面来看，这次开源市场的繁荣与上次网络泡沫时代相比，都有很多相似的地方。令人担心的是，很多创业公司（我曾听说过很多）启动资金都十分微薄，但他们都大胆承诺，如果他们的“愿景”被客户所接受，他们的产品就会很快流行开来。比如，Alfresco软件公司把很多市场上现有的开源内容管理软件的东西，结合到了自己的产品中，以节省成本。但是从积极的方面来看，开源应用已经实实在在存在，而且那些受到高度关注，并有风险投资支持的开源项目，也吸引到了开源社区中的核心人物来从事开发工作。但问题是，这些项目是否能够发展为企业级应用。由于牛顿和鲍威尔这样的行业专家（这些人知道怎样吸引风险投资，而且还知道如何使产品满足商业客户的需要）参与到开源投资中来，开源模式正在酝酿出像SugarCRM公司、Greenplum公司和Pentaho公司这样的赢利性企业。这些企业正在建立新一代的企业应用，用于管理Web内容、客户关系和企业资源。这些企业应用比商业应用软件更便宜，也可能更具有灵活性。而且，这些新应用的开发方式和商业模式，对传统软件开发与销售的模式可能将会产生非常重大的影响。


不过，不要将这些现象视作是一次普遍意义的“市场繁荣”，我们暂且称之为“开源的小小繁荣”吧。


Alfresco软件公司的成功，依靠的是管理团队的聪明才智，以及如何围绕免费下载的软件，提供有偿服务的能力。为了这个目标，公司必须从底层做起，他们必须去游说那些小企业来购买开源软件。这些企业要么宁愿自己建立文件共享系统，要么就是没有正式的内容管理，不愿花费大笔资金购买开源软件，只是凑合着做。牛顿对公司的定位很明确，就是通过提供使用方便和功能众多的产品，开拓出一个新的市场领域。比如，公司的产品具有安全性好、版本控制、元数据和全文搜索等多种功能，价格较高的软件包还有工作流功能。“有了我们的系统，你所要做的就是简单地拖曳内容而已。”公司主席兼首席技术官（CTO）牛顿表示。


Alfresco软件公司的创立，也显示了主流人才和资金怎样在追捧开源策略。牛顿和鲍威尔不仅带来了丰富的管理经验，而且还吸引了大量的投资，他们还从Accel合伙公司获得了200万美元的支持，这家风险投资公司曾经成功投资了RFID软硬件供应商萨维科技公司（Savi Technology）和开源服务器虚拟化软件供应商XenSource公司。


许多开源创业公司甚至不开发自己的软件，而是专门提供服务帮助企业实施流行的开源项目，并以此起家。Gluecode软件公司（Gluecode Software Inc.）就是选择了这样的发展道路，这家公司借助500万美元风险投资而创立，提供基于开源的Geronimo应用服务器的软件服务。不久前，国际商业机器公司（IBM）购买了这家不到20名员工的公司。目前，从IBM公司到太阳计算机系统公司（Sun），许多著名的软件供应商从未像现在这样积极地实施开源策略，尽管他们所涉足的领域千差万别。而他们的推动，正是促进目前开源运动发展的决定性因素。

目前，几乎任何一种商业软件都有着相对应的开源软件；同样，也有为数相当的供应商在为企业实施、使用和升级这些软件提供有偿服务。


尽管如此，商业科技经理们还是牢牢铭记着关于开源的那句老话：开源产品虽然是免费的，但是它就像一只捡来的小狗，一旦领进家门，你就要开始为它付出人工和金钱了。


对于像建筑管理企业Rudolf&Sletten公司这样的中型企业来说，要想部署某种套装软件来满足复杂的IT需求，开源应用软件几乎是他们的唯一选择。这家公司的IT经理萨姆·拉莫尼卡（Sam Lamonica）采用了GroundWork开源解决方案公司（GroundWork Open Source Solutions Inc.）提供的GroundWork Monitor网络管理应用程序，而没有选择BMC软件公司或者CA公司等提供的昂贵的商业软件。

GroundWork公司的系统基于开源的监视、Web门户和网络流量分析应用程序而创建，无需提前支付软件许可费用。“否则，我们就必须开始考虑那些大厂商的昂贵产品了。” 拉莫尼卡说。


牛顿表示，在内容管理市场每年25亿美元的收入中，超过一半来自于服务。也正是因为内容管理收入对服务的依赖性，给予了那些采用开源模式开发内容管理软件的企业巨大的机会。


当然，在功能全面性方面，新的开源产品还无法与现有的商业软件进行竞争。但开源的卖点主要是它对用户需求的快速响应度。因为在开源软件开发过程中，用户可以亲自参与，这样就可以准确地提供客户所需要的产品。由于用户直接参与到开源应用的升级过程当中，他们的需求就容易得以实现。与商业软件的模式相比，开源产品能更好地满足客户的需求，而商业软件的每次升级，用户则都需要付费。“比如Interwoven公司或Vignette公司提供的任何系统，以及这些系统所具备的最先进的功能，像语言翻译功能等。”ASPWatch公司的总裁约翰·哈尼（John Harney）举例说。哈尼所在的公司是一家为应用托管（Application-hosting）企业提供服务的咨询公司。不久前，他们对9个开源内容管理系统进行了评估，市场调研机构Cutter Consortium公司最近发布了这一评估结果。结果表明，大型跨国公司的需求来自不同国家、采用不同语言的数据。但大部分开源供应商无法提供这些功能，他们目前的用户也没有这方面的需求，或者说，他们的客户也不希望为这些高级功能付费。


Alfresco软件公司显然不会满足于简单的细分应用市场。和JBoss公司、MySQL公司类似，牛顿也抱着很大的雄心，他会将产品和服务提供给一些用户，并希望开源社区的程序员能够对Alfresco软件公司的产品提供技术支持，最终帮助公司开发出和市场上的领导厂商能够竞争的产品。专有软件供应商不会采取像开源社区那种紧跟用户需求的方式来跟客户交流。“它更多地成为了开发系统的人与使用系统的人之间的对话。”牛顿表示。


不过，要想平衡商业和社区的需求，并不容易。实际上，由于开源创业公司数量的不断增长，出现了一个突出的问题：这些企业的核心软件都是由无名的程序员编写的，至于牛顿们能否赚取更大利润，似乎跟他们毫无关系。在这种情况下，这批新兴的开源企业家能否带领企业获得成功呢？“大家贡献代码的方式还是有缺陷的，而且大部分开源软件商在开发产品与服务的时候，要想找到足够的外部工程师，并借助开源社区的力量，还是较为困难的。”布赖恩·贝伦多夫（Brian Behlendorf）表示。贝伦多夫是开源社区最成功的项目之一Apache Web服务器项目的共同创始人，以及协同开发工具制造商CollabNet公司的创始人兼CTO。“你会听到很多开发人员说，‘有些人已经在有偿开发某些复杂的软件，所以我要加入到他们当中。’”他补充说道。


由于新的商业开源混合型创业公司依靠开源程序员来贡献和测试代码、并提供新的创意，所以，对这些公司而言，如何赢得薄情寡义的开源社区的尊重至关重要。即使一家公司得到了大量的支持性编程人员，也很难管理他们完成商业目标。一些开源产品在项目中还聘用了核心开发人员，比如盈利性企业MySQL公司的开源数据库MySQL就是如此。但即便如此，也需要激活输入（Active Input）以及与客户的联动，以兑现相关软件对客户需求一定要快速反应的承诺。


显然，投资者并不畏惧这些针对开源商业模式的挑战。

Alfresco软件公司也不是唯一吸引风险投资的开源创业公司。提供开源IT监视程序的GroundWork公司，已经从梅菲尔德基金会（Mayfield Fund）以及迦南合伙人公司（Canaan Parnters）等风险投资商那里得到了1,150万美元的投资。提供客户关系管理软件和支持服务的SugarCRM公司，从德丰杰公司（Draper Fisher Jurvetson）和华登国际公司（Walden International）那里筹集到790万美元。近日，它又从新企业协会（New Enterprise Associates）获得1,877万美元的风险投资。


与.com时代的前辈们所不同的是，这些开源创业公司正在引进富有经验的经理人，其中很多人是专有软件领域的专家，而且这些企业正在进入商业计算市场的几乎所有领域。


SugarCRM公司的首席执行官（CEO）约翰·罗伯茨（John Roberts）表示，他创立公司的时候，正值CRM供应商互相整合，并开始扼杀创新的阶段。“进入这个市场的壁垒，并不在于一定要开发出更好的应用程序，而是厂商们宁可将70%的收入花在销售和市场营销上面。所以我们开始将眼光放在开源商业模式上。”罗伯茨表示。罗伯茨曾在商业CRM厂商E.piphany公司、美商宏道公司（BroadVision）和IBM公司担任过产品管理与销售等职位。


自从2004年初SugarCRM发布以来，该产品已经被下载数十万次之多，而且罗伯茨估计，这个软件被用在3.5万个工作场所。但和其他开源项目类似，它的用户多数是小企业：一般都是拥有3~20个用户的小企业。


在数据库领域，MySQL公司的MySQL数据库，比起开源PostgreSQL数据库，拥有一定的领先优势，尽管随着时间的变化，这种领先优势也会发生改变。

PostgreSQL的支持者甚至指望开辟出新的竞争领域。Greenplum公司在2005年4月发布了基于PostgreSQL的DeepGreen开源数据仓库软件测试版本。这家公司的CEO萨姆·M·萨卡其尼（Sam Mahamad Sakakini）曾在甲骨文公司（Oracle）从事了10年的销售和营销工作。萨卡其尼计划发布一个称为“DeepGreen MPP”的高端数据仓库系统，而且还在开发一些称为“Bizgres”的项目，来增强PostgreSQL在商业智能方面的应用，这也是另一个日趋升温的开源领域。


Greenplum公司的开源商业智能策略已经受到了Pentaho公司的挑战。这家公司成立于2005年6月，是Pentaho商业智能项目的组织者。Greenplum公司很多商业智能应用软件只是作为单独的报告，与分析模块和企业系统相分离，而Pentaho公司则将商业智能设计成企业流程的一部分，提供报告、分析、信息发送、数据挖掘和工作流功能。


Pentaho公司主席安德烈·布瓦维尔（Andre Boisvert）曾经是赛仕软件研究所有限公司（SAS Institute Inc.）主席兼COO、Oracle公司全球市场高级副总裁，同时他还曾是Congos公司和IBM公司的高级管理人员。Pentaho公司CEO理查德·达利（Richard Daley）以前是一家商业智能软件公司Keyola公司的CEO和创始人，这家公司2002年被劳森软件公司（Lawson）收购。更早以前，他是一家软件公司Appsource公司的CEO兼创始人，这家公司专攻前端商业智能查询、报告和分析产品。在1997年，Appsource公司被Arbor软件公司收购，而后者又在一年以后被海波龙软件公司（Hyperion Software）收购，并更名为海波龙解决方案公司（Hyperion Solution Corp.）。


达利和他的团队之所以选择开源之路来开发和发布Pentaho，是因为开源可以创造出完全不同于以往他所开发的那种商业智能平台。开源社区的资源，也意味着Pentaho公司可以利用已经免费发放的软件模块，而且得到全世界程序员的改进。“我们要做的大量事情，就是获取一些代码并将他们组合在一起，增加价值，最后返还给他们的原作者。”达利说。


达利对于Pentaho公司最终在商业智能软件市场称雄雄心勃勃。“我们会真正挑战那些市场领头羊，虽然这一过程还要经过3~5年的时间。”达利表示。达利之所以这样表示，是因为他认为商业智能软件是模块化的，它可以在某个时间替换掉某个组件。“人们不用抛弃任何现有的系统，但他们可以逐渐地迁移到开源平台。”达利这样说道。


对于Rudolf&Sletten公司的拉莫尼卡来说，部署了开源GroundWork软件，可能会给企业带来更多的开源应用，如果他发现了合适的应用，他就可以引入公司。“适合我们建筑行业的开源应用软件太少了。”拉莫尼卡说，“现在要么就是微软公司（Microsoft）的产品，要么你就别用，虽然微软公司的软件还是比较稳定的。”


为了使自己保持在开源市场开发的先锋地位，拉莫尼卡最近加入了SpikeSource公司的顾问委员会，这是一家提供软件与服务的创业公司，由前Mrimba公司高管金·波列斯（Kim Polese）组建，旨在通过测试、认证和支持服务来推动开源软件在企业内的应用。“我完全相信，要给用户提供不同的选择，这个产品就必须来自草根阶层。”波列斯表示。


拉莫尼卡正在寻找一种开源中间件，能够允许系统在应用程序间存储并转发信息。他希望加入SpikeSource公司的顾问委员会以后，能够通过与其他首席信息官（CIO）的关系网络，使自己看到更多的创意和不同的事物。


今天，在开源软件世界存在着一个巨大的鸿沟。有些软件已经运行在一些最尖端、高要求的IT运营领域，比如Linux操作系统和Apache Web服务器，而另外就是一些开源小公司，比如Pentaho公司、SugarCRM公司和Alfresco软件公司等，他们大多服务于20个小型企业用户。他们能取得实质性突破，并与现有的专有软件竞争吗？这个问题取决于企业客户是否愿意信任这样的模式，即这些软件部分控制在那些开源公司手中，他们有自己的经理人，并有赢利预期，而另一部分却没有任何控制。不过，已经有越来越多的企业家和风险投资家都纷纷下了赌注，因为他们认为客户将会接受这样的模式。

要点


一些创业软件公司正通过开源商业模式切入到最大的商业计算领域。
通过采取Linux、Apache以外的其他模式，资金充裕的创业公司正在积极开拓内容管理、商业智能、客户关系管理等领域。


开源创业公司正在吸引经验丰富的职业经理人，风险投资以及主流厂商的注意。
大部分开源应用目前主要提供给了小公司，但是MySQL和JBoss等项目的发展，显示了开源公司也有服务于大公司的潜力。

开源ERP？一家公司跨出了一大步


不久前，当Medal LP公司开始考虑为公司的企业管理软件升级的时候，它早早地就摈弃了50万美元的商业企业资源计划（ERP）软件包。这家公司是工业和医疗气体供应商法国液化空气集团美国LP公司（Air Liquide Group America LP）的子公司。


该公司希望能够找到一个更加简便的应用程序，比原有的Filemaker公司提供的软件更加简便。通过这个新的应用程序，公司能够实现会计、总账、发货和收货以及销售功能。尽管法国液化空气集团采用的是JD Edwards 公司的ERP系统 [注：这家公司已被甲骨文公司（Oracle）收购]，可是Medal LP公司“并不需要如此众多繁琐的功能”，Medal LP公司的IT经理安德鲁·霍洛夫卡（Andrew Horowka）认为。


霍洛夫卡转向了开源软件。


为什么？首先是成本。不过，其次，霍洛夫卡还希望新的程序在Linux和Mac OS X上都能够正常运行。经过调查，他选择了OpenMFG公司，这家公司开发了一个运行在PostgreSQL数据库上的ERP应用软件，“我到苹果公司网站的产品列表里察看了一番，还做了一些研究。”霍洛夫卡表示。


OpenMFG公司没有采用纯粹的开源模式。用户通过付费拥有许可证，这样就可以得到产品的源代码，而且他们可以对代码进行修改。但是对产品的所有修改成果都属于OpenMFG公司。“就像开源模式一样，我们把那些修改过的东西，再重新拿回来。但我们拥有这些代码的所有权，并且可以将他们应用在新的产品中。”这家公司的首席执行官（CEO）内德·莉莉（Ned Lilly）表示，“这就像是一个专有的通用公共许可证（GPL）。”


莉莉在1999年曾经为媒体集团Landmark通信公司（Landmark Communications Inc.）的风险投资部门效力，还经营过Landmark通信公司投资的一家创业公司，Great Bridge公司。她试图通过PostgreSQL数据库的商业化来创建一种新的商业模式。这个小公司最终没有活下来，但是莉莉尝到了开源市场的滋味。

开源对某些像ERP这样复杂的软件有意义吗？霍洛夫卡对此抱有怀疑。他研究了PostgreSQL社区并对数据库拥有这样一个开发人员社区的支持而感到满意。而且他曾经用过OpenMFG公司的代码，足以相信代码质量起码跟商业软件一样好，而且开源程序员“为自己的荣誉而工作，所以他们不会将垃圾代码留在程序里。”Medal LP公司在2005年年底前拥有15个OpenMFG系统的终端，而每年只需支付给供应商1.5万美元。


霍洛夫卡不希望OpenMFG公司、Compiere公司和其他开源ERP供应商改变竞争格局。但这不是他的最终目的。霍洛夫卡表示：“市场格局会影响我的营收状况。因为我们始终想要的是低成本的解决方案，用简单方法解决复杂问题。”
