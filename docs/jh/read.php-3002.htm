基本显示原理如下：

一．显示器的工作原理
目前在个人计算机上广泛使用的是採用阴极射线管（CRT）的光栅扫瞄显示器，我们在屏幕上所看到的顏色是由电子枪发出的电子束打在CRT屏幕背面的萤光层上的点形成的，通过控制点的亮度可以產生不同的顏色。电子束不断地从左到右、从上到下扫瞄整个屏幕，使屏幕显示出图案，电子束以大约每秒70次的速率在屏幕上重画这一图案，这个过程称為显示刷新或屏幕刷新，具体的扫瞄频率依赖於所用的显示适配器（又称為显示卡）。电子束从屏幕的左上角开始向右扫瞄，到达屏幕的右边缘后，电子束被关闭（水平断开），接著它又迅速地返回到屏幕的左边缘（水平回扫）开始进行下一行水平方向的扫瞄，在完成全部的水平方向的扫瞄后，电子束在屏幕的右下角结束，此时电子束被关闭（垂直断开），接著又迅速地返回到屏幕的左上角（垂直回扫），开始下一屏扫瞄。电子束就是这样週而復始地扫瞄整个屏幕。显示器在两种方式下工作：文本方式和图形方式，电脑游戏一般在图形方式下进行。

二．显示器的坐标系统
计算机屏幕上的坐标与我们通常使用的直角坐标系不同，坐标原点（0，0）在屏幕的左上角，向右是水平方向的坐标，向下是垂直方向的坐标，且坐标没有负值。

三．显示卡的结构
显示器上的显示卡负责将图形显示在屏幕上。显示存储器中存放著在屏幕上显示的图像数据，显示卡硬件不停地将显存中的内容显示在屏幕上。显示存储器实际上是安装在显示卡上的一块或几块大规模集成电路，其容量有1M、2M、4M、8M等，在DOS下我们可以访问的内存只有1MB空间（这就是DOS的局限性所在），地址从00000H到FFFFFH，这段内存根据用途又分為不同的块，系统分配给图形缓冲区（显示存储器）的地址在A0000H到BFFFFH之间，大小為128KB，其中，VGA佔用了A0000H到AFFFFH段，共64KB，这段地址是内存映射地址，供我们访问显示存储器用。在VGA 13H图形模式下，显示内存使用A0000H到AF9FFH的一段线性内存空间，每个字节表示一个点，对应屏幕上的一个像点，320*200的屏幕分辨率共需要64000个字节，刚好64KB，因為一个字节可以表示的最大整数值為256，所以每个像点就可以表示256种顏色。

四．设置视频模式
画图以前必须使屏幕工作在图形方式，这就要设置屏幕的视频模式。设置视频模式有许多种方法，其中调用视频BIOS功能是最简单的一种，通过调用BIOS中断0x10的服务程序，可以很方便地设置屏幕模式。调用方法是将值0放入ah寄存器，显示模式放入al寄存器中，然后调用int86()函数。

设置视频模式的函数：

void SetVideoMode(int mode)
{
union REGS r;
r.h.ah=0;
r.h.al=mode;
int86(0x10,&r,&r);
}

其中mode是视频模式。


五．在屏幕上画点
由於显示存储器是线性排列的，每个像点用一个字节来表示，所以对像点寻址非常容易，像点在显示内存中的偏移地址可由这个公式确定：y*320+x，其中，y是像点在垂直方向的坐标，x是水平方向的坐标，320是屏幕的宽度。有了像点的偏移地址，然后加上显示内存的首地址即可得到像点在显示内存中的绝对地址。只要将表示点的顏色值放到这个地址处，就可以在屏幕上画点了。


首先建立一个指针VideoBufferPtr，使它指向显示内存的首地址：

char far *VideoBufferPtr=( char far *)0xa0000000;

将这个指针加上像点的偏移地址，像点的最终地址就确定了，它等於：

VideoBufferPtr+y*320+x；

把顏色值color写到这个地址：

*(VideoBufferPtr+y*320+x)=color;

画点的函数：

void DrawPoint(int x,int y,unsigned char color)
{
*(VideoBufferPtr+y*320+x)=color;
}


六．显示字符

游戏经常在屏幕上打印出显示游戏状态的文本，使游戏者瞭解当前游戏的状态，这是游戏与游戏者进行交互的必不可少的。然而在图形模式下显示文本与在文本模式下有很大区别，在图形模式下文本必须使用基於画位图的方法来显示。这裡介绍一种简便的方法：利用计算机只读存储器中的ASCII字体数据。在计算机只读存储器（ROM）中固化有ASCII字体数据，这可在基址F000:FA6E上找到。我们只需瞭解数据是如何存储的，然后再得出存取该数据的算法，就可以用任意顏色在屏幕上用画位图的方法把字符画出来。在ROM中，字符按照ASCII字符编码的顺序放置，由於每个字符為8*8的点阵，所以每个字符佔据8个字节的存储空间，要找出某个字符，我们只需将这个字符的ASCII码与８相乘，然后将结果加到基址F000:FA6E上。由於字符用画位图的方法来显示，因此我们可以随意给它们指定顏色(前景或背景顏色)。在图形模式下，我们只需要两个函数就能够打印文本：一个用於显示单个字符，另一个用来显示字符串。


定义一个远指针指向ROM字符集的开始位置：

char far *RomCharPtr=(char far *)0xf000fa6e;

下面是在屏幕上打印字符和字符串的函数：

1．打印字符的函数。

void PrintChar(int cx,int cy,char c,unsigned char Fcolor,unsigned char Bcolor,int flag)
{
int offset,x,y;
char far *TempPtr;
unsigned char bit_mask;
TempPtr=RomCharPtr+(c<<3);
offset=(cy<<8)+(cy<<6)+cx;
for(y=0;y<8;y++)
{
bit_mask=0x80;
for(x=0;x<8;x++)
{
if((*TempPtr&bit_mask))
*(VideoBufferPtr+offset+x)=Fcolor;
else if(flag==1)
*(VideoBufferPtr+offset+x)=Bcolor;
bit_mask=(bit_mask>>1);
}
offset+=320;
TempPtr++;
}
}

说明：

cx,cy 是字符在屏幕上的坐标。

c 字符的ASCII码。

Fcolor,Bcolor 分别是字符的前景和背景顏色。

flag 打印标誌，当flag=1时显示字符的背景色，否则打印的字符具有透明效果。

2.打印字符串的函数。

void PrintString(int x,int y,char *string,unsigned char Fcolor,unsigned char Bcolor,int flag)
{
int index;
for(index=0;string[index]!=0;index++)
PrintChar(x+(index<<3),y,string[index],Fcolor,Bcolor,flag);
}

说明：

1.x,y 是字符串在屏幕上的坐标。

2.*string 字符串指针。

3.Fcolor,Bcolor 分别是字符串的前景和背景顏色。

4.flag 打印标誌，当flag=1时显示字符串的背景色，否则打印的字符串具有透明效果。


七．设置顏色寄存器

我们知道VGA显示卡具有显示256种顏色的能力，每种顏色能够用一个0-255之间的数值来表示，那麼这些数值与我们在屏幕上实际见到的顏色之间有什麼关係呢？其实这些数值只是VGA显示卡上的顏色寄存器的索引值，顏色寄存器裡才保存了屏幕上顏色的真实值。VGA显示卡上有一个包含256个单元的顏色寄存器（又称為调色板），每个单元由三部份组成，这三部份分别代表顏色中的红、绿、蓝三种成份（显示器就是用这三种成份来组成任何我们所看到的顏色），用三个字节表示，顏色寄存器一共有768个字节（3*256=768）。当我们要在屏幕上显示某种顏色时，显示卡硬件就根据顏色的索引值在顏色寄存器中查找，找到后再从相应单元中取出顏色值显示在屏幕上，这个过程与画家使用调色板相似，顏色寄存器相当於调色板，顏色寄存器中的单元相当於调色板上的色格，在色格中装有预先调好的顏色，当画家需要用某种顏色作画时，就从装有那种顏色的色格中把顏色取出来。例如，我们要显示顏色索引值為30的顏色，显示卡硬件就去查找顏色寄存器的第30单元，30单元位於距顏色寄存器首址3*30=90处（因為每个单元有三个字节），然后取出90处记录有红、绿、蓝三种成份的三个字节作為在屏幕上显示的色彩信号。但是实际上每个字节只用了六位来表示顏色，其它两位没用，这六位表示的数的值域為0-63，所以每种顏色（红、绿、蓝）成份具有64种亮度的表现能力，三种顏色成份组合共可以產生64*64*64=262,144种顏色(VGA 13H模式从这262,144种顏色中取出256种在同一屏幕上显示）。我们可以通过事先设置顏色寄存器的值来使用我们自己的顏色。
设置顏色寄存器有多种方法，如调用BIOS功能，但是这种方法速度比较慢，游戏设计中通常採用直接访问VGA显示卡的I/O端口的方法来快速设置顏色寄存器，我们只需访问四个I/O端口就可以完成设置顏色寄存器的工作。这四个端口分别是：
0x3c6、0x3c7、0x3c8和0x3c9。
端口0x3c6称為调色板屏蔽寄存器，用来屏蔽所要求的调色板寄存器的位，如果你在这个寄存器中放入0xff，你就可以通过调色板索引寄存器0x3c7和0x3c8（一个用於读，一个用於写）访问任何你希望访问的顏色寄存器，端口0x3c9称為调色板数据寄存器，红、绿、蓝三种成份就是通过它进行读写（顏色值要读或写三次）。


我们定义一个结构来方便处理顏色寄存器：

typedef struct RGB_COLOR
{
unsigned char red;
unsigned char green;
unsigned char blue;
}RGBColor,*RGBColorPtr;

结构中的red、green和blue变量用来保存顏色的红、绿、蓝三种成份。

设置顏色寄存器值的函数：

void SetPaletteRegister(int index,RGBColorPtr color)
{
outportb(0x3c6,0xff);
outportb(0x3c8,index);
outportb(0x3c9,color->red);
outportb(0x3c9,color->green);
outportb(0x3c9,color->blue);
}

获取顏色寄存器值的函数：

void GetPaletteRegister(int index,RGBColorPtr color)
{
outportb(0x3c6,0xff);
outportb(0x3c7,index);
color->red=inportb(0x3c9);
color->green=inportb(0x3c9);
color->blue=inportb(0x3c9);
}


八．在屏幕上画位图
计算机绘製图像通常採用一种称為位映射图（BITMAP）的图形处理方法进行，位映射图是一个矩形的点阵结构（二维矩阵），显示在屏幕上时，对应屏幕上一个矩形区域，组成位图的数据储存在内存中一段连续的区间。我们比较常见的位图文件有：BMP、PCX、GIF、JPG等。位图通常存储在外部文件中，使用以前必须将其从磁盘文件调入内存。下面介绍将一个256色PCX图形文件读入内存的方法：


1．定义PCX文件头结构：

typedef struct PCX_HEADER
{
char menufactrue; /* 厂家标识编号 0x0a */
char version; /* 文件版本编号 */
char packing_type; /* 压缩模式 */
char bits_per_pixel; /* 每点佔用的位数 */
int minx; /* 最小Ｘ坐标值 */
int miny; /* 最小Ｙ坐标值 */
int maxx; /* 最大Ｘ坐标值 */
int maxy; /* 最大Ｙ坐标值 */
int hres; /* 水平分辨率 */
int vres; /* 垂直分辨率 */
char palette[48]; /* 顏色调色板 */
char unused; /* 未使用 */
char bit_plance; /* 位平面个数 */
int bytes; /* 单一水平线佔用的字节数 */
int palette_type; /* 调色板类型 */
char unused2[58]; /* 未使用 */
}PCXHeader,*PCXHeaderPtr;

2．定义用来存放PCX图像数据的结构：

typedef struct PCX_PICTURE
{
int width;
int height;
char far *buffer;
RGBColor palette[256];
}PCXPicture,*PCXPicturePtr;

3．初始化图像数据的函数：

int InitPCX(PCXPicturePtr image,int w,int h)
{
unsigned size=w*h;
image->width=w;
image->height=h;
image->buffer=(char far *)farmalloc(size);
if(image->buffer==NULL) return 0;
return 1;
}

4．从外部文件读入数据的函数：

int LoadPCX(char *filename,PCXPicturePtr image,int flag)
{
FILE *fp;
unsigned num_bytes,count,size;
int index;
unsigned char data;
PCXHeader PcxHeader;
size=image->width*image->height;
if((fp=fopen(filename,"rb"))==NULL)
return 0;
fread(&PcxHeader,sizeof(PCXHeader),1,fp);
count=0;
while(count<=size)
{
data=fgetc(fp);
if(data>=192&&data<=255)
{
num_bytes=data-192;
data=fgetc(fp);
while(num_bytes-->0)
{
*(image->buffer+count)=data;
++count;
}
}
else
{
*(image->buffer+count)=data;
++count;
}
}
fseek(fp,-768L,SEEK_END);
for(index=0;index<256;index++)
{
image->palette[index].red=((fgetc(fp))>>2);
image->palette[index].green=((fgetc(fp))>>2);
image->palette[index].blue=((fgetc(fp))>>2);
}
fclose(fp);
if(flag==1)
for(index=0;index<256;index++)
SetPaletteRegister(index,(RGBColorPtr)&image->palette[index]);
return 1;
}

其中参数flag用来指明调入文件的同时是否设置顏色寄存器（flag=1设置）。

5．画位图的函数：

void DrawImage(int x,int y,int width,int height,char far *image)
{
int i,j;
for(i=0;i<height;i++)
for(j=0;j<width;j++)
{
if(*image!=0&&(x+j)>=0&&(x+j)<320&&(y+i)>=0&&(y+i)<200)
DrawPoint(x+j,y+i,*image);
image++;
}
}

x,y是图像在屏幕上的左上角坐标，width,height是图像的宽度和高度，image是指向内存中图像的指针。

我们对if(*image!=0&&(x+j)>=0&&(x+j)<320&&(y+i)>=0&&(y+i)<200)语句进行一下分析：

*image!=0用来检查所画的顏色值是否是透明色，如果是，则不画出来，这样我们就可以画出有透明效果的图像，即透过图像可以看到背景，透明色通常取值0，也可以用其他的顏色值表示。
(x+j)>=0&&(x+j)<320&&(y+i)>=0&&(y+i)<200语句用来判断所画点的坐标是否超出屏幕显示的范围，这样可以画出具有裁剪效果的图像，如图像的一部份在屏幕外。

这个函数并不是最快的，因為它要执行width X height次判断，更快的函数请看VGA13H函数库中的绘图函数。

VGA显示系统

显示硬件基础

　　进行TC256色图形编程时，是直接与显卡硬件在打交道。那麼，就让我们从显卡谈起吧。
　　显示卡(adapter)是一块插在PC主机的电路版。一般显示卡由寄存器、存储器（显示RAM和ROM BIOS）、控制电路三大部分组成。随著PC机的发展，PC机的显示系统经歷了由CGA、EGA、VGA、SVGA到现在的VESA系列的发展过程。
　　在早期的显示系统中，CGA、EGA等显示卡配置的显示器是严格配套的，一种显卡只与一种显示器匹配使用。随著技术的发展，出现了可变频的显示器，它使用自动跟踪技术，同一显示器可以适应CGA、EGA、VGA等各种显示卡。这使得我们用TC操作显示卡模拟VESA工作模式成為可能。
　　顏色和灰度是衡量显示系统的又一重要参数。最初的彩色显示系统CGA只能显示固定的16种顏色，到EGA时可以使用64种顏色种的16种顏色，而发展到VGA时可以使用256K种顏色中的16种顏色，现在的显示卡甚至可以支持到32位真彩色并使用所有顏色。顏色与灰度的发展受显示内存的限制。在16色模式中，一个像素点只需要log2 16=4位内存即半个字节；而在256色模式中，一个像素点则要占log2 256=8位即一个字节；在16位真彩色模式下，一个像素要佔用整整两个字节。所以，在CGA高分辨率方式下，只能同时使用两种顏色，因為640*200*1/8 = 16000（byte）接近16k。而到了VGA，实现640*480分辨率16种顏色时，由於一个像素有16种顏色，最少需要占4位，因此，需要内存640*480*1/8 = 153600（byte）≒ 150k。


此外，RAM的扩大还受到PC机中留作显示用的地址空间的限制，一般不超过64K。因為，当使用大於64K的地址空间时，通过PC机访问显示缓冲区的控制就变得较為复杂了（使用换页机制）。
　　现在的显卡可以兼容多种显示模式，因此，显卡在固化的ROM BIOS中為每一种模式分配了一个代号。用户在使用PC机编程时，直接输入该代号，再通过显示卡的BIOS调用即可使用此种模式。
　　通常，显示模式号<0x14的是标準模式，其它的為非标準模式，非标準模式因显示卡的不同而不同。

显示缓冲区与顏色定义

　　PC机的显示卡上的RAM存储器中的数据按照显示器显示格式进行存储。我们知道 ，计算机只能以二进制方式存储数据，每位有两种状态（0与1）。对於单色显示器，内存中只需存放一张表，表中每一位对应屏幕上一个像素点，该位為1则表示该点是亮点。而对於彩色显卡来，要表示屏幕上像素点的信息，仅用一為就不够了。对於显示16色的显示模式，就需4位定义一种顏色。到了EGA显示卡时，已经开始有了显示卡兼容性。在VGA模式时由於顏色较多，分辨率提高带来点数的增加，所需要的显存增大，从而出现了一种以「彩色位平面」的存储结构来表示顏色信息。CGA400、CEGA、CH、VGA均採用彩色位平面的存储结构。一般VGA的基本配置有256K的显示缓冲区（BANK A）。当支持16种以下的顏色时，使用存储位平面结构，此时256K的显示缓冲区被分位4个64K的存储位平面，屏幕上的一个点，由存储位平面的各一位组合后表示，最多表示16种顏色。当支持256种顏色时，要使用线性内存结构，4个存储位平麵线性链接，形成256K的线性内存（Linear system）。
　　在使用16色和256色模式显示时，需要一张顏色表，以将显存的数据「翻译」為屏幕上的点信息。所谓顏色表，就是我们通常所指的「调色板」。当显示器要显示屏幕上的一个点时，先由显示卡将显存中的数据读出，然后对照顏色表，得到一组RGB顏色信息，然后调整显示器的射线管，在屏幕相应位置显示一个点。当显存中所有的点都显示后，就得到我们所看到的图像。用户还可以根据自己显示的需要，更改顏色表的RGB对应值，得到我们自定义的顏色。
　　在显示模式到了真彩色级别时，由於显示内存中存储的已经是像素点的RGB信息，因此调色板就变得没有意义。因此，在真彩色模式中，不再需要调色板。

视频BIOS ROM

　　显示卡都有自己的专用视频BIOS支持。此BIOS是视频控制程序，固化在ROM中，成為显卡的一部分，显示卡上的视频BIOS功能要比PC的视频BIOS强大的多，它除了支持CGA、EGA、VGA等各标準显示模式外，还支持各种专用模式和针对显存的专用操作，可以独立於CPU单独处理显存图像，比如2D变换、3D环境计算等等非常复杂的运算。一般VGA视频BIOS的入口地址在C000H——CFFFFH之间。

BMP是bitmap的缩写形式，bitmap顾名思义，就是位图也即Windows位图。它一般由4部分组成：文件头信息块、图像描述信息块、顏色表（在真彩色模式无顏色表）和图像数据区组成。在系统中以BMP為扩展名保存。
　　打开Windows的画图程序，在保存图像时，可以看到三个选项：2色位图（黑白）、16色位图、256色位图和24位位图。这是最普通的生成位图的工具，在这裡讲解的BMP位图形式，主要就是指用画图生成的位图（当然，也可以用其它工具软件生成）。
　　现在讲解BMP的4个组成部分：

1.文件头信息块

0000-0001：文件标识，為字母ASCII码「BM」。
0002-0005：文件大小。
0006-0009：保留，每字节以「00」填写。
000A-000D：记录图像数据区的起始位置。各字节的信息依次含义為：文件头信息块大小，图像描述信息块的大小，图像顏色表的大小，保留（為01）。


2.图像描述信息块


000E-0011：图像描述信息块的大小，常為28H。
0012-0015：图像宽度。
0016-0019：图像高度。
001A-001B：图像的plane总数（恆為1）。

001C-001D：记录像素的位数，很重要的数值，图像的顏色数由该值决定。
001E-0021：数据压缩方式（数值位0：不压缩；1：8位压缩；2：4位压缩）。
0022-0025：图像区数据的大小。
0026-0029：水平每米有多少像素，在设备无关位图（.DIB）中，每字节以00H填写。
002A-002D：垂直每米有多少像素，在设备无关位图（.DIB）中，每字节以00H填写。
002E-0031：此图像所用的顏色数，如值為0，表示所有顏色一样重要。

3.顏色表

　　顏色表的大小根据所使用的顏色模式而定：2色图像為8字节；16色图像位64字节；256色图像為1024字节。其中，每4字节表示一种顏色，并以B（蓝色）、G（绿色）、R（红色）、alpha（32位位图的透明度值，一般不需要）。即首先4字节表示顏色号0的顏色，接下来表示顏色号1的顏色，依此类推。

4.图像数据区

　　顏色表接下来位為位图文件的图像数据区，在此部分记录著每点像素对应的顏色号，其记录方式也随顏色模式而定，既2色图像每点占1位（8位為1字节）；16色图像每点占4位（半字节）；256色图像每点占8位（1字节）；真彩色图像每点占24位（3字节）。所以，整个数据区的大小也会随之变化。究其规律而言，可的出如下计算公式：图像数据信息大小=（图像宽度*图像高度*记录像素的位数）/8。

然而，未压缩的图像信息区的大小。除了真彩色模式外，其餘的均大於或等於数据信息的大小。这是為什麼呢？原因有两个：
　　1.BMP文件记录一行图像是以字节為单位的。因此，就不存在一个字节中的数据位信息表示的点在不同的两行中。也就是说，设显示模式位16色，在每个字节分配两个点信息时，如果图像的宽度位奇数，那麼最后一个像素点的信息将独佔一个字节，这个字节的后4位将没有意义。接下来的一个字节将开始记录下一行的信息。
　　2.為了显示的方便，除了真彩色外，其他的每中顏色模式的行字节数要用数据「00」补齐為4的整数倍。如果显示模式為16色，当图像宽為19时，存储时每行则要补充4-(19/2+1)%4=2个字节（加1是因為裡面有一个像素点要独佔了一字节）。如果显示模式為256色，当图像宽為19时，每行也要补充4-19%4=1个字节。
　　还有一点我要申明，当屏幕初始化為16或256色模式时，一定要设置调色板或修正顏色值，否则无法得到正确的图像顏色。
置256色图形模式

　　要使用256色图形模式，使用TC的界面模式是不行的，因為它為VGA16色的文本模式，要使用256图形模式，则要调用显卡BIOS进行图形界面初始化。具体是调用10H的显示中断，将功能号AH置為0，子功能号AL置為要使用的模式号，以调用IBM兼容显卡的320×200的256色模式為列，程序如下。
#include "dos.h"
void init256(int Vmode)
{ union REGS r;
　r.h.ah=0;
　r.h.al=Vmode;
　int86(0x10,&r,&r);
}
main()
{ init256(0x13);
　getch();
　init256(0x3);
}
运行后，屏幕进行了一次闪烁，实际上，这是屏幕显示模式切换引起的。切换模式后，屏幕一片黑暗，因為切换模式后，未对屏幕进行任何操作。接下来，我将演示在256色模式下对显卡的操作。

访问显存

　　初始化得到了256色图形模式的屏幕，接下来要做的就是向屏幕显示图像了。在显示图像之前，我要介绍一个很重要的far指针。也许你在16色VGAHI模式裡用过，那就是内存中的图形模式显存映像指针0xa0000000，说它是映像指针，是因為显示器只与显示卡直接打交道，所以要更改屏幕内容还需更改显存内容。因此，内存中提供一块64k的区域，通过显示中断可以使它映射為显存的不同区域，更改该内存中的值即更改了相应映射区域的显存。让我们先试一下下面的程序：（注意：运行时，最好将内存模式设置為large，否则可能出错，以后我不再提示）
#include "dos.h"
main()
{ long i;
　char far *p=MK_FP(0xa000,0000);
　union REGS r;
　r.x.ax=0x13;
　int86(0x10,&r,&r);
　for(i=0;i<320*200L;i++)*(p+i)=i/320;
　getch();
　r.x.ax=0x3;
　int86(0x10,&r,&r);
}


看见了什麼，你会惊诧地发现：多麼丰富的色彩！DOS模式下的256色显示实现了！

显示卡换页

　　前面将过由於内存只提供64k的内存进行显存操作，在320×200的256模式下尚可访问所有的屏幕像素点。但是在320×200以上的256模式下就不行了，因為在此模式下，比如在640×480的256模式下，存储一屏所有的像素点需要640×480byte=300k的存储单元。比如使用以下程序：
#include "dos.h"
main()
{ long i;
　char far *p=0xa0000000;
　union REGS r;
　r.x.ax=0x5f;/*TNT2、GeForce2的640×480的256色模式均使用该模式号*/
　int86(0x10,&r,&r);
　for(i=0;i<640*400L;i++)*(p+i)=(i/640)%256;
　getch();
　r.x.ax=0x3;/*恢復為16色文本模式*/
　int86(0x10,&r,&r);
}

一、关於虚拟屏幕

　　虚拟屏幕技术是当今电脑游戏编程中採用的一项先进技术，但此其编程内幕却未见在电脑或软件期刊、网页等媒体上有多少透露。本人不揣冒昧，愿将自己实践的结果公诸於世，供广大游戏编程爱好者参考。

　　所谓虚拟屏幕，就是在内存中分配一块内存，并将其当成显存进行绘图操作，当绘图完成时，就将其复製到显存中。

　　使用虚拟屏幕技术有什麼好处呢？

　　一、操作方便,速度快。在游戏中，有大量的图块存取操作。但在高清晰度下，显存是分页操作的，特别是行内分页的显示模式下进行图块存取更是困难。由於虚拟屏幕的物理位置是在内存中的，所以不存在分页问题，而且在操作速度上要比显存快上二倍多。

　　二、虚拟屏幕大小不受显示分辨率的影响。在内存中，可以创建任意大小的虚拟屏幕，在显示时只拷贝一部分到显存即可。

　　三、有利於播放动画。在内存中，我们可以创建多个虚拟屏幕，各存入一帧图像，然后按顺序进行显示，即可以实现动画的快速播放。

　　四、卷轴动画。所谓卷轴动画，即创建几层虚拟屏幕，一层叠加一层。《真侍魂》就是利用这一技术的代表作游戏。

二、虚拟屏幕技术的适用范围

　　虚拟屏幕的应用主要有三种：
　　一、大屏幕。常见的大型游戏中，比如《仙剑奇侠传》、《红色警戒》都是先将整个地图创建成一个超大的虚拟屏幕，在虚拟屏幕上进行操作。显示时就将要显示的一部分拷贝到显存中显示出来。而只将提示信息、菜单等操作在真实屏幕中进行。

　　二、多层屏幕卷轴动画。这一方面也主要应用於游戏中。著名的2D格斗游戏《街霸》、《真侍魂》和国產RPG《江湖》就充分的利用了这一技术，在《真侍魂》中，程序创建了八层虚拟屏幕，分别是远背景(云,月,星星等)、近背景、裁判、第一层可砍景物(柳生场地右边的竹子)、人物、动物(狗,鹰,猴)和气功(旋风,火等)、第二层可砍景物(柳生场地左边的竹子)、血槽和气量表。对八个层次的虚拟屏幕分别进行操作，然后进行组合，组合时通过不同的移动速度就可以產生很强的立体感(远景移动快;近景移动慢)。

　　三、多个屏幕。这一技术在MacOS、Windows(2.0及更高版本)、OS/2等图形界面操作系统中没有不使用的。拿常见的Windows来说，Windows為每个窗口建立一个虚拟屏幕，等有了窗口刷新的消息时，才将虚拟屏幕中的内容在真实屏幕上显示出来。

三、关於VESA、32位DOS平台和Watcom C++

　　VESA是当今普遍使用的显示卡编程接口，可以说在市场上已经没有了不支持VESA标準的显示卡。所以，我们在编写显示程序时只需编写一个VESA显示程序即可获得最大的兼容性。

　　在16位实模式的DOS平台下，能够直接操作的内存只有640k，而现在的内存16MB已是相当普及了。如果使用EMS或XMS，则要进行二次数据存取，将大大影响操作速度。倘若在编程时採用DPMI 32位保护模式技术进行编程，虽然可以越过640k基本内存的限制，但由於程序仍然是16位的，让CPU在32位保护模式下运行16位程序，运行速度将会变的相当慢。然而，使用Watcom C/C++ 32位模式进行编程，则可以直接实现由16位到32位、640k到4GB的飞跃。当今的一些著名游戏都是使用Watcom C/C++进行编写的，例如《金庸群侠传》、《阿猫阿狗》、《红色警戒》等等。

四、编程注意事项

　　由於Watcom C/C++编出的程序是32位的，所以在编程时同Borland C/C++有所不同。要注意以下几点：

　　一、数据长度不同。在Borland C/C++3.1中，int数据是16位的。而在Watcom C/C++中，int数据则是32位的。

　　二，内存地址不同。32位保护模式下的内存(包括显存在内)都是经过重新影射过的。同实模式下的内存地址不同。如果使用实模式的显存地址进行读写，那后果则是无法预料的。所以在进行显存读写操作时，必须对显示地址进行转换之后才能使用。

　　三、中断调用方式不同。在32位保护模式下，不能使用int86、int86x进行中断调用。要使用int386、int386x进行中断调用。

五、程序的特点


　　这是一套為开发游戏而编写的及其商业化的显示驱动程序。其显示速度超快无比。在Pentium 133、800x600x16位增强色模式、中文Windows98的DOS窗口下，全屏刷新速度高达每秒40屏左右，最高时可达每秒45屏。若是在纯DOS方式下速度更快。而DirectX6.0中的DirectDraw6.0只能达到每秒鐘27屏左右。这主要是由於Windows不允许软件直接操作硬件造成的，用DirectX编写的游戏，窗口的虚拟屏幕中其实根本就没有写入任何东西。DirectX越过了Windows的兼管，直接向显存中写入数据来实现快速显示。这种手法真是让人好笑，DirectX让Windows程序又回到了DOS状态。因為DOS程序从来都是直接操作硬件的，所以速度要比Windows程序快的多。

　　说它商业化，不仅是因為它速度快，主要是因為它是基於游戏需要，完全為2D游戏所设计编写的。所以只提供了几个简单的函数：画点(原本也没有)、画水平线、画垂直线、画矩形、画块；而没有提供画线、画圆等函数。由於篇幅限制，省略了图形变换、显示中英文字符、显示窗口、图块操作等函数。(如若需要可向作者索取)

六、程序使用方法

　　本套显示驱动程序有二个文件，头文件Svga.h和源程序文件Svga.cpp。使用时将Svga.h连入程序，编译时连Svga.cpp一起编译即可。在程序中，首先要像DirectDraw一样建立一个控制指针，再通过此指针进行图形操作，要显示时再执行Redraw()函数进行屏幕刷新。

video=_video;Cls(); //创建后备指针并清屏
}
VIEW::~VIEW() //本类析构函数，自动执行
{
free(_video); //释放虚拟屏幕所用内存
SetVideo(3); //初始化屏幕為文本状态
}
void VIEW::SetVideo(uvar16 mode) //设置显示模式,mode為显示模式
{
union REGS i;
if(mode>=0x100) //如果是VESA显示模式
{
i.w.bx=mode;
i.w.ax=0x4F02;
int386(0x10,&i,&i);
return;
}
else //如果是标準显示模式
{
i.w.ax=mode;
int386(0x10,&i,&i);
return;
}
}
void VIEW::VideoPage(uvar8 pn) //选择显存页函数,pn為页号
{
union REGS i;
i.w.bx=0;
i.w.dx=pn;
//如果您的显卡VESA标準版本较低，一页是4k而不是64k的，将会显示不正常，请将上一行改為：
//i.w.dx=pn*16;
i.w.ax=0x4F05;
int386(0x10,&i,&i);
return;
}
void VIEW::Redraw() //屏幕刷新函数
{
uvar8 i;
uvar16 *VIDEO=(uvar16 *)(0xA000<<4); //显存指针
video=_video; //恢復虚拟屏幕指针
for(i=0;i<14;i++)
{
VideoPage(i); //选择显存页面
memcpy(VIDEO,video,65536); //从虚拟屏幕拷贝64k数据到显存
video+=32768; //虚拟屏幕指针后移
}
VideoPage(14); //选择最后一页
memcpy(VIDEO,video,42496); //写入最后一页数据
video=_video; //恢復虚拟屏幕指针
}
inline void VIEW::Cls() //将顏色置0(黑色)并清屏
//内置式函数
{
memset(_video,color=0,960000); //将整个虚拟屏幕全部写入0
}
